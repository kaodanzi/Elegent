# Codex-maxxing 中文润色增强版设计

## 目标

基于现有的中文版 PDF，生成一版更接近正式中文出版物的润色增强版。

本版目标：

- 保留原 PDF 的页面结构、图片、图形元素与整体版式。
- 图片中的英文不翻译。
- 将正文、标题、目录、页眉页脚中的中文改写为更自然的出版物中文。
- 统一高频术语，避免同一概念在不同页面出现多种译法。

## 风格原则

- 整体风格采用“自然出版物风”。
- 表达追求自然、凝练、可连续阅读，不机械贴近英文句法。
- 在不改变原意的前提下，允许对语序、词性和句式做适度重写。
- 标题优先简洁有力，正文优先流畅清楚。

## 术语策略

采用“产品名保留英文、通用概念译成中文”的统一规则。

核心约定如下：

- `Codex`：保留英文 `Codex`
- `thread`：译为“线程”；在需要强调产品交互语境时可用“对话线程”
- `durable thread`：统一译为“持久线程”
- `voice input`：统一译为“语音输入”
- `steering`：统一译为“引导”
- `memory`：统一译为“记忆”
- `vault`：统一译为“记忆库”
- `computer and browser use`：统一译为“计算机与浏览器操作”
- `remote control`：统一译为“远程控制”
- `thread automations`：统一译为“线程自动化”
- `goals`：统一译为“目标”
- `side panel`：统一译为“侧边栏”
- `diffs`：视上下文译为“差异”或“变更对比”，优先保证中文顺畅
- `repo` / `repository`：统一译为“代码仓库”
- `open loops`：统一译为“未闭环事项”

## 润色范围

本次润色覆盖全篇文字层，但按内容类型分层处理。

### 1. 封面与目录

- 对主标题、副标题做出版物风格改写。
- 目录项采用短标题，优先保证一眼可懂。
- 页眉页脚中的重复标题同步统一。

### 2. 章节标题与大字引语

- 保留原稿气质，采用更凝练、更像中文书面标题的表达。
- 避免直译造成的生硬词组，如“耐用的螺纹”“把记忆变成你可以回顾的东西”等。

### 3. 正文段落

- 以自然中文表达重写句子。
- 合并不必要的英文式断裂语序。
- 保留事实、例子、逻辑关系和产品操作含义。

### 4. UI 标签与结构性短语

- 文件树、标签、按钮式短语优先保持短、稳、明确。
- 对会影响排版的短文本，优先使用长度可控的固定译法。

## 实现方案

采用现有“保留原版式、覆盖文字层”的 PDF 生成流程，不改图片与底图。

处理步骤：

1. 在现有翻译缓存基础上，新增人工润色映射层。
2. 建立可复用的术语表与重点短语白名单。
3. 先处理封面、目录、章节标题、页眉页脚等高可见文本。
4. 再处理正文段落，将机器直译改为人工润色版本。
5. 复用现有脚本重新生成 PDF。
6. 抽查关键页面，确认无明显溢出、重叠、错位或术语不一致。

## 质量标准

- 同一术语在全篇中保持一致，除非上下文必须区分。
- 标题应像中文标题，而不是英文逐词替换。
- 正文应可顺畅朗读，不出现明显“机翻腔”。
- 不出现错误概念，例如：
  - `thread` 误译为“螺纹”
  - `Codex` 误译为“法典”
  - `vault` 误译为“拱顶”或“保管库”
- 不因润色造成新的版面破坏。

## 风险与边界

- 由于中文长度和英文长度差异较大，少数标题或短块可能需要更凝练的译法来适配版面。
- 极少数视觉密集页面可能需要在“完全自然”与“版式稳定”之间做轻微折中。
- 本次不改图片内容，不对图片中的英文做覆盖翻译。

## 交付物

- 一份“中文润色增强版” PDF
- 更新后的润色脚本与术语映射逻辑
- 可复用的术语与短语映射数据，便于后续继续迭代
